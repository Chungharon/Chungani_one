## Problem

Businesses needed a modern, fast ecommerce experience without the overhead of
maintaining legacy platforms. The goal was to build a lightweight, fully custom
store that could be deployed globally and scaled without infrastructure concerns.

## Approach

The stack was chosen for developer velocity and production reliability:

- **Next.js App Router** for server-rendered product pages with fast static generation
- **Stripe** for payments — Stripe Checkout handles PCI compliance out of the box
- **Tailwind CSS** for a design system that stays consistent at scale
- **Vercel** for zero-config global deployment with edge caching

The product catalog is statically generated at build time. Cart state lives
client-side in `localStorage` with a React context layer. Checkout redirects
to a Stripe-hosted session, keeping sensitive payment data off our servers entirely.

```bash
git clone
cd ecommerce-store
npm install
npm run dev
```

## Key Features

- **Stripe Payments** — Checkout sessions with webhooks for order confirmation
- **Product Catalog** — Grid layout with image optimisation via `next/image`
- **Shopping Cart** — Persistent cart with quantity controls and subtotal
- **User Authentication** — Session-based auth with protected order history
- **Order History** — Full order timeline with shipment tracking links

## Outcome

- Lighthouse performance score: **97**
- Time to first byte under **120ms** on Vercel's edge network
- Zero payment incidents in production — Stripe webhook reliability handled all edge cases
- Deployed and live within **2 weeks** of starting the build
