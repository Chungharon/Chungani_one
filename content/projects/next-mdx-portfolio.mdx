## The Concept

Most portfolio templates are over-engineered — too many dependencies, too much
JavaScript, too little control. This design explores the opposite: a site that
is fast by default, fully owned by the developer, and content-managed through
plain MDX files checked into git.

No CMS. No database. Just markdown with superpowers.

## Design Decisions

**MDX over a headless CMS** — content lives in the repository. Every post and
project is a `.mdx` file with frontmatter. Writing is done in any text editor.
Versioning is git. This trades a visual editing UI for complete ownership and
zero vendor lock-in.

**File-based routing** — Next.js App Router maps the `content/` directory
structure directly to URLs. Adding a new post is as simple as creating a file.

**Tailwind Typography** — the `@tailwindcss/typography` plugin handles all
prose styling. Dark mode is a single class toggle. No custom CSS needed for
readable long-form content.

**No client JavaScript for reading** — post and project pages are fully static.
React only ships to pages that need interactivity (contact form, theme toggle).

## What It Looks Like in Practice

```mdx
---
title: My Post Title
summary: A one-line description.
publishedAt: '2026-01-01'
---

## Introduction

Write in **Markdown** and use <CustomComponents /> where needed.
```

That file becomes a fully styled, SEO-optimised, statically served page at
build time.

## Takeaway

The best portfolio is one you'll actually keep updated. Removing friction from
the writing process — no login, no editor, just a file — is the real design win here.
